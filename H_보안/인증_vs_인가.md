# 인증(Authentication) vs 인가(Authorization)

인증은 “누구인가?”, 인가는 “무엇을 할 수 있는가?”를 묻는다. 두 개념을 명확히 구분하면 보안 설계가 쉬워진다.

---

## 1. 인증
- 사용자의 신원을 확인하는 절차.
- 방법: 비밀번호, OTP, MFA, OAuth2 Login, SSO, 장치 인증 등.
- 구현 시 고려: 페더레이션(IdP), 비밀번호 정책, 계정 잠금, 로그인 감사 로그.

## 2. 인가
- 인증된 사용자가 어떤 리소스에 접근 가능한지 결정.
- 모델: RBAC(역할 기반), ABAC(속성 기반), 정책 엔진(Opa, Casbin).
- 정책 변경/검토 프로세스 필요.

---

## 3. 체크리스트
- [ ] 인증/인가가 다른 서비스/모듈로 분리되어 있는가?
- [ ] MFA 적용 여부, 위험 기반 인증 도입 검토?
- [ ] 권한 변경(승인/로그) 절차가 명확한가?
- [ ] 접근 거부/허용 결과를 감사 로그에 남기는가?

---

## 4. 실습 아이디어
- **실습 1**: OAuth2 + RBAC를 적용한 API를 구현.
- **실습 2**: OPA 정책을 작성해 ABAC를 시연.
- **실습 3**: 인증 실패/인가 실패 로그를 분석해 공격 패턴 탐지.
