# 데이터 암호화 · 해싱 · 시크릿 관리

데이터 보호는 저장 시와 전송 시 모두 필요하다. 이 문서는 암호화/해싱/시크릿 관리의 기본을 다룬다.

---

## 1. 암호화 vs 해싱
- **암호화**: 복호화 가능. 대칭키(AES), 비대칭키(RSA).
- **해싱**: 단방향. 비밀번호 저장 시에는 Bcrypt/Argon2 사용.
- **전송 중 보호**: TLS/HTTPS, 내부 서비스 간 mTLS.

## 2. 시크릿 관리
- 환경 변수/코드에 시크릿을 하드코딩하지 않는다.
- Secret Manager(AWS Secrets Manager, HashiCorp Vault), KMS 등을 사용.
- 시크릿 접근을 최소 권한으로 제한하고, 로테이션 자동화.

## 3. 데이터 분류
- PII/PHI 등 민감 데이터는 암호화/마스킹/토큰화.
- 동의/규제(GDPR, HIPAA 등)에 맞춰 보관 기간과 활용 범위를 정의.

---

## 4. 체크리스트
- [ ] 저장 데이터 암호화(Encryption at Rest)와 전송 암호화(Transport Encryption)가 적용되어 있는가?
- [ ] 비밀번호/토큰을 안전한 해시 알고리즘으로 저장하는가?
- [ ] 시크릿을 중앙에서 관리하고 로테이션 절차를 갖고 있는가?
- [ ] 암호화 키 접근/사용 로그를 감사하고 있는가?

---

## 5. 실습 아이디어
- **실습 1**: Vault/KMS로 API 키를 저장하고 애플리케이션에서 동적으로 로딩.
- **실습 2**: Bcrypt/Argon2 해싱을 적용해 Rainbow Table 공격을 방어.
- **실습 3**: S3/KMS 조합으로 저장 데이터 암호화를 적용해 본다.
