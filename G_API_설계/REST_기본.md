# REST 설계 기본 (메서드, 상태 코드, 버저닝)

RESTful API를 설계할 때 지켜야 할 기본 원칙을 정리한다.

---

## 1. 리소스/엔드포인트 설계
- 명사형 경로 사용 (`/users`, `/orders/{id}`).
- 계층 구조 표현 (`/orders/{id}/items`).
- 컬렉션(복수), 개별 리소스(단수) 구분.

## 2. HTTP 메서드 & 상태 코드
- `GET`: 조회, `POST`: 생성, `PUT/PATCH`: 수정, `DELETE`: 삭제.
- 상태 코드: 200 OK, 201 Created, 204 No Content, 400 Bad Request, 401/403, 404, 409, 500 등.
- 에러 응답은 일관된 JSON 형식(`code`, `message`, `traceId`)으로 제공.

## 3. 버저닝 전략
- URL 버전(`/v1/users`), 헤더 버전(`Accept: application/vnd.app.v1+json`), 쿼리 파라미터.
- 주요 변화(스키마 수정, 필드 제거) 시 새로운 버전 도입.
