# 트랜잭션 ACID 특성

ACID(Atomicity, Consistency, Isolation, Durability)는 데이터베이스 트랜잭션이 제공해야 하는 4가지 성질이다. 이 문서는 각 특성을 설명하고, 실제로 어떻게 검증/적용하는지 안내한다.

---

## 1. Atomicity (원자성)
- 트랜잭션 내 연산은 “전부 성공하거나 전부 실패”해야 한다.
- 롤백/커밋을 명확히 사용하고, 예외 발생 시 반드시 트랜잭션을 종료한다.

## 2. Consistency (일관성)
- 트랜잭션 전후로 데이터가 비즈니스 규칙을 위반하지 않도록 한다.
- 외래 키, 체크 제약, 트리거, 도메인 로직으로 보장한다.

## 3. Isolation (격리성)
- 동시에 실행되는 트랜잭션 간 상호 간섭을 최소화한다.
- Read Uncommitted, Read Committed, Repeatable Read, Serializable 등 격리 수준으로 조정.

## 4. Durability (내구성)
- 커밋된 데이터는 시스템 장애가 발생해도 유지되어야 한다.
- WAL(Write-Ahead Logging), 복제, 스냅샷, 백업으로 보장한다.

---

## 5. 체크리스트
- [ ] 롤백/커밋이 누락된 부분이 없는가?
- [ ] 데이터 무결성 제약(키, 제약 조건, 트리거)을 정의했는가?
- [ ] 격리 수준이 비즈니스 요구와 부하에 맞는가?
- [ ] 백업/복구 테스트를 정기적으로 수행하는가?

---

## 6. 실습 아이디어
- **실습 1**: 동일한 트랜잭션을 다양한 격리 수준에서 실행하며 Dirty Read/Non-repeatable Read 등을 체험.
- **실습 2**: 장애 상황(서버 강제 종료)을 시뮬레이션해 커밋 데이터가 유지되는지 확인.
- **실습 3**: 외래 키/체크 제약 조건을 의도적으로 위반해 Consistency가 어떻게 보장되는지 실험.
