# 클라우드 핵심 서비스 비교 (AWS/GCP/Azure)

주요 클라우드의 핵심 컴퓨팅/스토리지/네트워크 서비스를 비교해 선택 기준을 잡는다.

---

## 1. 컴퓨팅
- **AWS EC2 / GCP Compute Engine / Azure VM**: 기본 IaaS.
- **컨테이너 서비스**: ECS/EKS, GKE, AKS.
- **서버리스**: Lambda, Cloud Functions, Azure Functions.

## 2. 데이터베이스
- RDS / Cloud SQL / Azure SQL Database.
- DynamoDB / Cloud Bigtable / Cosmos DB.
- Redshift / BigQuery / Synapse.

## 3. 네트워크 & 스토리지
- S3 / GCS / Blob Storage.
- CloudFront / Cloud CDN / Azure CDN.
- VPC, Cloud VPN, Direct Connect / Interconnect / ExpressRoute.

---

## 4. 선택 기준
- 서비스 수준 협약(SLA), 지역/AZ 가용성.
- 비용 모델(온디맨드, 예약, 스팟).
- 팀 역량과 기존 인프라와의 호환성.
- 관리형 서비스 지원 여부(Managed DB, 메시지 큐, Observability).

---

## 5. 체크리스트
- [ ] 비용 예측을 위해 Cost Explorer/Billing Alert를 설정했는가?
- [ ] 멀티 리전/멀티 클라우드 전략이 필요한가?
- [ ] 보안/규제 요구 사항(데이터 지역화 등)을 만족하는가?
- [ ] 각 클라우드의 관리형 서비스에 대한 Runbook을 갖고 있는가?

---

## 6. 실습 아이디어
- **실습 1**: 동일한 서비스를 AWS/GCP/Azure에 배포해 비용/관리 차이를 비교.
- **실습 2**: 서버리스(FaaS)를 사용해 이벤트 기반 애플리케이션을 구축.
- **실습 3**: CloudWatch/Stackdriver/Azure Monitor를 이용해 기본 지표를 수집.
