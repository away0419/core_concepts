# 대용량 트래픽 처리 · 코드 리뷰 · 문서화 · 레거시 개선

기술 역량을 정량적으로 증명하려면 실무 경험을 체계적으로 기록해야 한다.

---

## 1. 대용량 트래픽 대응 사례 정리
- 트래픽 패턴, 병목 지표, 어떤 조치를 했는지(스케일링, 캐시, 쿼리 튜닝).
- 전/후 성능 수치(P95, QPS, 비용)와 교훈을 기록.

## 2. 코드 리뷰 문화
- 리뷰 템플릿(Why, What, Test)을 정의하고, 자동화된 체크(CI, Lint)를 선행.
- 리뷰 피드백은 기록으로 남겨 향후 베스트 프랙티스 문서에 반영.

## 3. 기술 문서 작성
- ADR(Architecture Decision Record), Runbook, Postmortem, API 스펙 등 문서를 버전 관리.
- 신규 팀원이 빠르게 온보딩할 수 있도록 다이어그램/예시 포함.

## 4. 레거시 개선
- AS-IS 분석 → 목표 정의 → 단계적 리팩터링/재작성.
- Feature Flag/Strangler Pattern으로 리스크 최소화.
- 개선 전/후 지표를 측정해 성과를 수치화.

---

## 5. 체크리스트
- [ ] 주요 프로젝트마다 성능/안정성 개선 지표를 기록했는가?
- [ ] 코드 리뷰와 문서화가 프로세스에 내재화되어 있는가?
- [ ] 레거시 개선 계획과 진행 상황을 대시보드로 공유하는가?

---

## 6. 실습 아이디어
- **실습 1**: 실제 프로젝트 경험을 템플릿(문제-조치-결과)으로 문서화.
- **실습 2**: 코드 리뷰 가이드라인을 만들고 팀에 적용.
- **실습 3**: 레거시 모듈을 Strangler Pattern으로 교체하는 PoC 작성.
