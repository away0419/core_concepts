# 시스템 장애 대응 · 재발 방지 · 성능 병목 분석

실무에서 가장 중요한 역량은 장애를 빠르게 복구하고, 같은 문제가 반복되지 않도록 개선하는 것이다.

---

## 1. Incident Response 절차
1. 감지(Alert) → 담당자 호출(PagerDuty 등).
2. 상황 파악(대시보드/로그/트레이스).
3. 완화 조치(롤백, 트래픽 분산, 임시 스케일링).
4. 커뮤니케이션(내부 채널, Status Page).
5. Postmortem 작성 및 재발 방지 계획.

## 2. Postmortem 구성
- 사건 요약, 타임라인, 사용자 영향, 근본 원인(RCA), 교훈, 행동 항목(Action Items).
- 비난 금지(Blameless) 문화.

## 3. 성능 병목 분석
- Latency Budget, Flame Graph, Profiling, DB 모니터링, 캐시 히트율 등을 총괄적으로 분석.

---

## 4. 체크리스트

## 5. 실습 아이디어
- **실습 1**: Chaos Engineering으로 장애를 주입하고 대응 흐름을 연습.
- **실습 2**: 실제 장애 사례를 바탕으로 Postmortem 문서를 작성.
- **실습 3**: Flame graph/프로파일러로 병목 함수를 찾아 최적화.
