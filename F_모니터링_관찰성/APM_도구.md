# APM 도구의 역할 (NewRelic / Datadog / Pinpoint 등)

APM(Application Performance Monitoring)은 애플리케이션 내부 동작을 추적해 병목을 찾아낸다.

---

## 1. 핵심 기능
- 코드 수준 트레이싱 (메서드, SQL, 외부 호출).
- 에러 추적 및 스택 트레이스 수집.
- 메트릭/로그/트레이스 통합 대시보드.
- 배포 마커, 릴리즈별 성능 비교.

---

## 2. 대표 도구
- **NewRelic, Datadog, Dynatrace**: SaaS형, 다양한 언어/인프라 지원.
- **Pinpoint, SkyWalking**: 오픈소스, Java/모노리스 환경에 강점.
- **AWS X-Ray, Azure Monitor**: 클라우드 통합형.

---

## 3. 도입 체크리스트
- [ ] 에이전트 설치로 인한 오버헤드를 허용할 수 있는가?
- [ ] 샘플링 비율과 보존 정책을 비용에 맞게 조정했는가?
- [ ] PII/민감 정보를 마스킹하거나 수집하지 않도록 설정했는가?
- [ ] 대시보드/경보 템플릿을 팀 표준으로 마련했는가?

---

## 4. 실습 아이디어
- **실습 1**: 샘플 애플리케이션에 APM 에이전트를 설치하고, 느린 트랜잭션을 식별.
- **실습 2**: 배포 전/후 지표를 비교해 성능 회귀를 탐지.
- **실습 3**: APM Alert를 설정해 P95 지연이 상승할 때 PagerDuty/Slack으로 알림.
